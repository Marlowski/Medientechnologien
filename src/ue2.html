Bits pro Pixel gibt an, wie viel Speicher für ein Pixel zu verfügung gestellt wird, also wie genau der Farbton gespeichert werden kann. Hier: 24 Pixel bzw. 8-bit für je ein {R,G,B} Wert

Dateigröße: Die Dateigröße lässt sich mit 640*480*3 = 921.600 Berechnen (in Bytes). Die Angabe im Header beträgt 921.654, was darauf schließen lässt, dass 54 Bytes für sonstige Informationen, wie Metadaten verwendet wird.

2.4)

Erkläre, wie bei Bild- und Videoaufnahmen Aliasing entstehen kann. Ähnlich wie bei der 1. Übung am Audiosignal wollen wir auf das Bild ein Down-Sampling anwenden und sehen, wann dabei Aliasing auftritt. Beim Audiosignal haben wir jedes zweite Sample verworfen und damit die Abtastfrequenz halbiert. Beim Bild verringern wir die räumliche Auflösung, indem wir die Zahl der Bildzeilen bzw. Bildspalten verringern. Dabei wollen wir das Bild allerdings nicht verkleinern - was normalerweise die Folge wäre - sondern einfach die ungeradzahligen Spalten durch den Inhalt der geradzahligen Spalten ersetzen, d.h. Spalte 1 wird durch Spalte 0 ersetzt, Spalte 3 durch Spalte 2 etc. und analog für die Zeilen. Wenn das Ursprungsbild nun sehr feine Strukturen aufweist, wie in machen Testdateien, können Aliasing-Verzerrungen sichtbar werden. Achte darauf, dass stets Rot, Grün und Blau getrennt manipuliert werden müssen.

Zeitliches Aliasing in einem Video kann ensteht aufgrund einer zu niedrigen Bildwiederholfrequenz. Das am weitverbreitetste Beispiel ist das Rad, welches bei einem eingentlich vorfährtsfahrenden Fahrzeug wirkt als ob es rückwärts rollt oder sogar still steht. Das liegt daran, dass ein sich zu schnell drehendes Rad das Abtasttheorem verletzt. Die Drehung des Rades kann man als Signalfrequenz betrachten & wenn sich dieses zu schnell dreht kann afgrund der zu niedrigen Bildwiederholfrequenz nicht mehr korrekt abgetastet werden und es entstehen oben genannte Aliasing-Effekte.

Wie schon in Aufgabe 2.1 angedeutet, können Bilder genauso wie Frequenzen als Schwingungen abgebildet werden. Sollte also das Abtasttheorem dann nicht eingehalten werden, entstehen sogenannte "Moire-Muster". Der Grund dafür ist, dass das Pixelraster, auf welchem abgetastet wird, zu grobmaschig ist und den schnellen Hell-Dunkel-Wechsel nicht korrekt erfassen kann.

Wir wollen nun bmp_io so modifizieren, dass wir die Bitzahl reduzieren können. Dazu können wir z.B. alle RGB-Werte durch eine Potenz von 2 teilen (Integer-Division ohne Rest). Damit das resultierende Bild nicht dunkler wird als das Original, kompensieren wir die Operation durch Multiplikation mit derselben Zweierpotenz. Zu beachten: Der Datentyp hat nach wie vor eine Bittiefe von 24Bit (8bit pro Farbe

int reduced_bits = 5;
for (int y = 0; y < bmp.image.getHeight(); y++) {
 for (int x = 0; x < bmp.image.getWidth(); x++) {
  PixelColor currentPx = bmp.image.getRgbPixel(x, y);
  double potenz = Math.pow(2, reduced_bits);
  currentPx.r /= potenz;
  currentPx.g /= potenz;
  currentPx.b /= potenz;
  currentPx.r *= potenz;
  currentPx.g *= potenz;
  currentPx.b *= potenz;
 }
}

3.3)

Modifiziere dein Programm noch einmal so, dass auch das Differenzbild zwischen Original und bitreduziertem Bild, das heißt das Fehlerbild ausgegeben, werden kann. Beachte dabei, dass bei der Differenzbildung negative und positive Werte auftreten können. Diese müssen wieder in den Wertebereich 0...255 transformiert werden, d.h. auf den Mittelwert 127 gebracht werden (wie?). Dabei musst du auch bedenken, dass z.B. bei 1 Bit Reduzierung der Fehler nur 0, 1 oder -1 wäre. Dieser Wertebereich wäre viel zu klein, als dass man ihn in der 8bit-Grafik noch sehen könnte. Daher muss der Fehler durch Multiplikation mit einer 2er Potenz verstärkt werden. In anderen Worten: Hat man vorher durch 2^n geteilt, sollte man das Differenzsignal vor dem Abspeichern mit 2^(8-n-1) multiplizieren. So ist sichergestellt, dass der Verstärkungsfaktor mit der Anzahl der gelöschten Bits kleiner

// bitreduzierung differenz
int reduced_bits = 3;
final int bitsPerColor = 8;
for(int y = 0; y < bmp.image.getHeight(); y++){
 for(int x = 0; x < bmp.image.getWidth(); x++){
  PixelColor currentPixel = bmp.image.getRgbPixel(x, y);
  PixelColor copyPx = new PixelColor(currentPixel.r, currentPixel.g, currentPixel.b);
  copyPx.b /= Math.pow(2, reduced_bits);
  copyPx.r /= Math.pow(2, reduced_bits);
  copyPx.g /= Math.pow(2, reduced_bits);

  copyPx.b *= Math.pow(2, reduced_bits);
  copyPx.r *= Math.pow(2, reduced_bits);
  copyPx.g *= Math.pow(2, reduced_bits);

  currentPixel.b -= copyPx.b;
  currentPixel.r -= copyPx.r;
  currentPixel.g -= copyPx.g;

  copyPx.b *= Math.pow(2, bitsPerColor - reduced_bits - 1);
  copyPx.r *= Math.pow(2, bitsPerColor - reduced_bits - 1);
  copyPx.g *= Math.pow(2, bitsPerColor - reduced_bits - 1);

  copyPx.b = copyPx.b / 2 + 127;
  copyPx.r = copyPx.r / 2 + 127;
  copyPx.g = copyPx.g / 2 + 127;

  bmp.image.setRgbPixel(x, y, copyPx);
 }
}
